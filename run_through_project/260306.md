# DB 설계를 이용한 REST API 설계
## DRF With MySQL
- 장고는 기본적으로 SQLite라는 파일 형식 데이터베이스 사용. 
- 하지만 팀 프로젝트를 위해 MySQL로 변경해서 사용하기로 함.
- 이번에 서버까지 연결하는 건 배보다 배꼽이 큰 일이라 더미 데이터를 이용하기로.
- fixtures로 json형태의 데이터를 저장하는 방법 할 예정
### 데이터 전처리
- 파일 읽을 때 상대경로 쓰면 작업위치에 따라 경로가 달라질 수 있다
- 프로젝트 파일의 settings.py에서 BASE_DIR이라는 변수를 만들어서 프로젝트의 최상위 경로를 저장해놓음
- BASE_DIR을 이용해서 파일 경로를 지정하면 작업 위치에 상관없이 항상 올바른 경로로 파일을 읽을 수 있음
### MySQL 연동
- 장고에서 MySQL을 사용하려면 mysqlclient라는 패키지를 설치해야 함
- requirements.txt에 추가되어 있어 pip install -r requirements.txt로 설치 가능
- settings.py에서 DATABASES 설정을 MySQL로 변경

- 데이터베이스는 장고에서 만들게 하지 말고 sql로 직접 만들기
- settings.py의 데이터베이스 설정 냅다 퍼블릭으로 올리면 안됨

- djrestauth에서 업데이트로 회원가입 성공시 상태코드 204에서 201로 변경됨

- 회원가입 오류 뜨는 이유는 로컬에서 request해서 연결이 잘 안될때도 있기 때문
- 회원가입 했는데 first_name, last_name이 안들어가 있다! 이는 RegisterSerializer에서 필요로 하지 않기 때문.. -> RegisterSerializer 커스터마이징 필요 -> accounts에서 serializers.py 만들어서 RegisterSerializer 커스터마이징하기
- 커스텀 이후 유효성 검사 get_cleaned_data()에서 first_name, last_name이 빠져있어서 추가해주기
- 커스텀이 완성되었다면 settings.py에서 REST_AUTH_REGISTER_SERIALIZER에 커스텀한 RegisterSerializer의 경로를 지정해주기

## fixtures
- django가 데이터베이스로 가져오는 방법을 알고 있는 데이터 모음
- 데이터는 데이터베이스 구조에 맞춰 작성 되어있음
- 목적 : 초기 데이터 제공
- 프로젝트 진행할 때 깃에 데이터베이스가 올라가지 않기 때문에 fixtures로 데이터를 관리하는 것이 좋음
- dumpdata와 loaddata로 json파일 확인해서 어떻게 데이터를 작성해야 하는지 확인하기
- 한글 데이터 dumpdata시 -Xutf8 옵션 사용 or 메모장으로 json파일 열어서 다른 이름으로 저장, 인코딩을 UTF-8로 선택해서 저장하기
