# Basic SQL

## DML(Data Manipulation Language)
- 테이블 내의 데이터를 추가, 수정, 삭제, 조회하는 명령어
- 데이터 조회는 기능이 많아 별도의 DQL(Data Query Language)로 분류하기도 함
### INSERT
- 테이블 레코드 삽입
- `INSERT INTO table_name (column1, column2, ...) VALUES (value1, value2, ...);`
- VALUES는 괄호, 쉼표로 여러 값 입력 가능
- ```sql
    INSERT INTO table_name 
    (column1, column2, ...) 
    VALUES 
    (value1, value2, ...),
    (value1, value2, ...); 
  ```
- created_at과 updated_at 같은 경우는 NOW() 함수로 현재 시간 입력 가능
  - NOW()로 날짜나 시간 입력 양식 신경 덜 쓰고 편하게 입력 가능
- 컬럼명 생략 시 순서 맞춰서 값 입력만 하면 되긴 함. 하지만 권장하지 않음
  - 어지간하면 명시적으로 작성할 것. 
### UPDATE
- 테이블 레코드 수정
- ```SQL
    UPDATE table_name 
    SET column1 = value1, column2 = value2, ...
    [WHERE 
      condition];
  ```
- WHERE 절로 수정할 레코드 지정 조건 작성 가능
- WHERE 절 생략 시 테이블의 모든 레코드가 수정됨
### DELETE
- 테이블 레코드 삭제
- ```SQL
    DELETE FROM table_name 
    [WHERE 
      condition];
  ```
- WHERE 절로 삭제할 레코드 지정 조건 작성 가능
- WHERE 절 생략 시 테이블의 모든 레코드가 삭제됨
### DELETE와 TRUNCATE의 차이
- TRUNCATE와 DELETE 모두 테이블의 모든 레코드를 삭제 가능
- DELETE는 데이터를 삭제하기 때문에 pk값이 초기화되지 않음
- TRUNCATE는 테이블 자체를 DROP후 재생성하기 때문에 pk값이 초기화됨
- PK 초기화 안되는 건 인덱스 관점에서 봤을 때 비효율일 수 있지만 기본값일 뿐이고 순서가 중요한 게 아니라 전체적으로 봤을 때 비효율인 건 아님
## DQL
### 사전 준비
- USE database_name; 명령어로 사용할 데이터베이스 선택
### SELECT
- 테이블에서 데이터 조회
- ```SQL
    SELECT column1, column2, ...
    FROM table_name;
  ```
- *로 모든 컬럼 조회 가능
- AS 키워드로 별칭 지정 가능
  - 컬럼명이나 테이블명 대신 별칭으로 조회 가능
  - `SELECT column_name AS alias_name FROM table_name;`
- 간단한 연산도 가능
  - `SELECT column1 + column2 AS total FROM table_name;`
## Filtering Data
### DISTINCT
- 조회 결과에서 중복된 레코드 제거
- SELECT 바로 뒤에 작성
- `SELECT DISTINCT column_name FROM table_name;`
- 고유한 값을 선택하려는 하나 이상의 필드 지정
### WHERE
- 조회 시 특정 검색 조건 지정
- ```SQL
    SELECT column1, column2, ...
    FROM table_name
    WHERE condition;
  ```
- FROM 절 다음에 작성
- 조건은 비교연산자(=, <>, >, <, >=, <=)와 논리연산자(AND, OR, NOT) 사용 가능
  - 다른 프로그래밍 언어와 다르게 SQL에서는 등호(=)로 비교, != 대신 <> 사용
- 문자열 비교 시 작은따옴표(')로 감싸야 함
- NULL은 비교연산자 사용 불가, IS NULL, IS NOT NULL로 작성해야 함
  - NULL이 SQL에서 특별한 값이라 여러 특별한 연산자를 가짐
### LOGICAL 0PERATORS
- AND(&&), OR(||), NOT(!)
### Comparison Operators
- =, <>, >, <, >=, <=
- BETWEEN ... AND ..., IN (...), LIKE, IS 등
- BETWEEN ... AND ...: 범위 지정
  - `SELECT column_name FROM table_name WHERE column_name BETWEEN value1 AND value2;`
  - value1 <= column_name <= value2 라고 쓰면 안됨. 첫 조건만 만족하면 TRUE로 간주하기 때문
- IN (...): 값 이 특정 목록 안에 있는지 확인
- LIKE: 특정 패턴에 일치하는지 확인
  - Wildcard와 함께 사용
  - %: 0개 이상의 문자열과 일치하는지
  - _: 정확히 1개의 문자와 일치하는지
  - `SELECT column_name FROM table_name WHERE column_name LIKE 'A%';` -> A로 시작하는 모든 값 조회
  - `SELECT column_name FROM table_name WHERE column_name LIKE 'A____';` -> A로 시작하고 뒤에 공백을 포함하여 정확히 4개의 문자가 있는 데이터
  - 'A%'와 'A %'는 다름. 'A%'는 A로 시작하는 모든 값 조회, 'A %'는 A로 시작하고 뒤에 공백이 있는 모든 값 조회
- IS : NULL값인지 아닌지 확인
### 연산자 우선순위
1. 괄호
2. NOT
3. 비교 연산자
4. AND
5. OR
- 우선순위 고려 없이 작성하면 의도치 않은 동작 발생
- 실수할 것 같으면 괄호 사용해서 명시적으로 우선순위 지정하는 것이 좋음

## MySQL Built-in Functions
### 문자형 함수
- CONCAT(str1, str2, ...): 문자열 연결
- TRIM([[LEADING | TRAILING | BOTH] remstr] target_str): 문자열 왼쪽/오른쪽/양쪽 특정 문자 제거
  - LEADING: 왼쪽에서 remstr 제거, TRAILING: 오른쪽에서 remstr 제거, BOTH: 양쪽에서 remstr 제거
  - remstr 생략 시 공백 제거(주로 사용)
- REPLACE(str, from_str, to_str): 문자열 내에서 특정 문자열을 다른 문자열로 대체
- LOCATE(substr, target_str [,pos]): target_str에서 substr의 위치 반환, 없으면 0 반환
  - substr이 target_str에 여러 번 나타나는 경우 첫 번째 위치 반환
  - pos 생략 시 target_str의 처음부터 검색, pos 지정 시 pos 위치부터 검색
### 숫자형 함수
- ABS(x): x의 절대값 반환
- MOD(x, y): x를 y로 나눈 나머지 반환 (n % m 과 동일)
- POW(x, y): x의 y 제곱 반환
- CEIL(x): x의 올림값 반환
- FLOOR(x): x의 내림값 반환
- ROUND(x[, d]): x를 소수점 d자리까지 반올림하여 반환
### 날짜형 함수
- CURDATE(): 현재 날짜 반환
- CURTIME(): 현재 시간 반환
- NOW(): 현재 날짜와 시간 반환
- DATE_FORMAT(date, format): 날짜를 원하는 format형태로 변환
  - format 예시: '%Y-%m-%d' -> '2024-06-30', '%M %d, %Y' -> 'June 30, 2024'
  - %y: 2자리 연도, %Y: 4자리 연도, %m: 2자리 월, %M: 월 이름, %d: 2자리 일, %H: 24시간제 시간, %h: 12시간제 시간, %i: 분, %s: 초
  - 필요할 때 검색해서 사용하면 됨
### NULL 관련 함수
- IFNULL(expr1, expr2): expr1이 NULL이면 expr2 반환, 아니면 expr1 반환
- NULLIF(expr1, expr2): expr1과 expr2가 같으면 NULL 반환, 아니면 expr1 반환
- COALESCE(expr1, expr2, ...): 첫 번째 인자부터 순서대로 확인해 NULL이 아닌 값을 반환, 모두 NULL이면 NULL 반환
  - 순서가 중요
  - ```SQL
    SELECT NAME COALESCE(NULLIF(GNP, 0), 'GNPOld', 'No data') AS gnp_data
    FROM country;
    ```
  - GNP가 0이면 'GNPOld' 반환, GNP와 GNPOld가 모두 NULL이면 'No data' 반환, GNP가 0이 아니면 GNP 반환
## Sorting Data
### ORDER BY
- 조회 결과 정렬
- FROM 절 다음에 작성
- 오름차순이 기본값, 내림차순은 DESC 키워드 사용
  - 오름차순도 명시적으로 적어주는 것 권장
- ```SQL
  SELECT column_name 
  FROM table_name 
  ORDER BY column_name [ASC|DESC];
  ```
- 여러 컬럼으로 정렬 가능. 이 경우 먼저 나온 컬럼 정렬 후 그 안에서 다른 컬럼으로 정렬
  - ```SQL
      SELECT column1, column2 
      FROM table_name 
      ORDER BY column1 ASC, column2 DESC;
    ```
- NULL값이 존재할 경우 오름차순 정렬에 NULL이 먼저 출력됨. 이 경우 NULL을 마지막에 출력하고 싶으면 IS NULL 조건 추가
  - ```SQL
      SELECT column_name 
      FROM table_name 
      ORDER BY column_name IS NULL, column_name ASC;
    ```
### LIMIT
- 조회 결과에서 반환할 레코드 수 제한
- FROM 절 다음에 작성(ORDER BY 다음)
- 하나 또는 두 개의 인자 사용(0 혹은 양의 정수)
- ```SQL
  SELECT column_name 
  FROM table_name 
  LIMIT [offset,] row_count;
  ```
- offset 생략 시 0으로 간주, row_count는 반환할 최대 레코드 수

## Grouping Data
### AGGREGATE FUNCTIONS(집계 함수)
- 값에 대한 계산을 수행하여 단일 값을 반환하는 함수. GROUP BY와 많이 사용
- COUNT(expr): expr이 NULL이 아닌 레코드 수 반환
- COUNT(*): 레코드 수 반환(NULL 포함)
- SUM(expr): expr의 합계 반환(NULL은 무시)
- AVG(expr): expr의 평균 반환(NULL은 무시)
- MAX(expr): expr의 최대값 반환(NULL은 무시)
- MIN(expr): expr의 최소값 반환(NULL은 무시)
- STDDEV(expr): expr의 표준편차 반환(NULL은 무시)
- VARIANCE(expr): expr의 분산 반환(NULL은 무시)
### GROUP BY
- 레코드를 그룹화하여 요약본 생성 (집계 함수와 함께 사용)
- FROM, WHERE 절 다음에 작성
- ```SQL
  SELECT column_name, AGGREGATE_FUNCTION(column_name)
  FROM table_name
  WHERE condition
  GROUP BY column_name;
  ```
- GROUP BY절 뒤에 그룹화할 컬럼 나열
- GROUP BY절에 명시된 컬럼은 SELECT절에도 포함되어야 함
- 집계 함수로 계산된 컬럼을 SELECT절에 포함한 후 WHERE로 조건 지정하면 오류 발생. 이 경우 HAVING절로 조건 지정해야 함
  - HAVING절은 GROUP BY 없어도 사용 가능하지만(집계 함수는 있어야 함) 일반적으로 GROUP BY와 함께 사용. 
  - 
    ```SQL
      SELECT column_name, AGGREGATE_FUNCTION(column_name)
      FROM table_name
      GROUP BY column_name
      HAVING AGGREGATE_FUNCTION(column_name) condition;
    ```
  - WHERE절이 GROUP BY절보다 먼저 실행되기 때문에 집계 함수로 계산된 컬럼을 WHERE절에서 사용할 수 없음

### SELECT statement 실행 순서
1. FROM 절: 데이터 소스 결정
2. WHERE 절: 레코드 필터링
3. GROUP BY 절: 레코드 그룹화
4. HAVING 절: 그룹 필터링
5. SELECT 절: 결과 컬럼 계산 및 선택
6. ORDER BY 절: 결과 정렬
7. LIMIT 절: 반환할 레코드 수 제한
