# DFS
## 비선형 자료구조
- 비선형구조인 트리, 그래프의 각 노드를 중복되지 않게 전부 방문하는 것.
- 비선형구조는 선형 구조와 달리 선후 연결 관계를 알 수 없어 특별한 방법 필요
- 두 가지 방법 존재
  - 깊이 우선 탐색(DFS, Depth First Search)
  - 너비 우선 탐색(BFS, Breadth First Search)
## 깊이 우선 탐색(DFS)
- 루트 노드에서 출발, 한 방향으로 갈 수 있을 때까지 깊이 탐색, 더 이상 갈 수 없으면 가장 마지막에 만났던 갈림길로 돌아와 다른 방향의 노드로 탐색 반복
  - 모든 노드 방문하는 순회 방법
- 가장 마지막에 만났던 갈림길의 노드로 되돌아가서 다시 깊이 우선 탐색 반복 -> 재귀적, 후입선출 스택 자료구조 활용
### 구현
- 트리 탐색 재귀적 구현 의사코드
  ```
  DFS(v)
    v 방문
    for(v의 모든 자식 노드 w) {
      DFS(w)
    }
  end DFS()
  ```
- 교재의 dfs 예시 옆의 호출 스택은 dfs 함수가 호출될 때마다 쌓이는 함수 호출 정보를 나타냄
### DFS - 그래프
- 시작 정점에서 출발, 인접한 정점 방문, 더 이상 방문할 인접 정점이 없으면 가장 마지막에 방문한 갈림길 간선이 있는 정점으로 되돌아가서 다시 인접 정점 방문 반복
- 그래프는 사이클이 존재할 수 있으므로 방문한 정점을 다시 방문하지 않도록 방문 처리 필요
  - 방문할 수 있는 노드에 방금 방문한 노드가 포함될 수 있기 때문
- visited 배열 활용
- 그래프에서도 간선에 있는 애를 자식이라고 부르던가?
- 트리와 그래프의 dfs 의사코드 차이
  ```
  DFS(v, graph, visited)
    v 방문
    visited[v] = true
    for(v의 모든 인접 정점 w) {
      if(visited[w] == false) {
        DFS(w, graph, visited)
      }
    }
  end DFS()
  ```
- 이후에는 경로에 따른 비용도 생각해야 함
- 직접 스택 구현해서 반복문으로 구현 가능
  - 해당 함수는 그래프 인접 리스트를 사용한다

- 재귀랑 루프랑 어째 방문 순서가 조금씩 다르다. 로직이 다르긴 해서 그런듯
  - target 스택에 b부터 들어가고 c가 들어가서 c가 먼저 나옴
  - 재귀는 a에서 b로 갔다가 b에서 d로 갔다가 d에서 e로 갔다가 다시 b로 돌아와서 c로 감
- 재귀랑 루프랑 방문 순서가 다를 수 있지만 뭐가 더 빠를지를 고민하는 건 의미없음. 그건 인접 리스트를 어떻게 그리느냐에 따라 달라짐. 그래서 빅-오 표기법을 쓰는것. 대충 비슷한 성능이니까


## 실습 강의 메모
### 그래프 코드 
- 나중에 그래프에서 일부러 간선 몇 개를 배제하고 진행하는 경우 발생할 것.
- 0번노드를 무조건 시작점으로 잡진 않을수도
- 비선형 구조에서 시작점을 잘 설정하는 게 중요
- 그래프 그리는 데 집중. 인접행렬, 인접 리스트 만드는 연습 할 것.
- 그래프를 전부 순회하려면 모든 노드에서 갈 수 있는 모든 노드 방문을 반복헤야함. -> for문을 그렇게 돌릴 순 없고 재귀적으로 구현해야함
- 방문 했는지 여부에 대한 체크 필요 -> 방문처리 배열
- 동시 조사가 아니고 하나씩 누적 비용 생기면서 조사 => 뭔말이지?
- 옆으로 퍼져나가는 게 아니라 깊이 파고 들어가는 게 DFS
- 0번에서 1번을 갈 수 있다는 걸 확인하면 1번으로 옮겨서 다시 1번에서 갈 수 있는 노드로 이동
- 깊이 '우선' 탐색인거지 깊이 탐색은 아님
- 재귀, 스택/for문 사용해서 구현해볼거임
### dfs 트리구조 순회
- 노드가 많아질수록 인덱스만으로는 다음 대상을 찾기 어려움
=> 노드 객체를 만들어서 연결리스트로 관리
### dfs 그래프구조 순회
- 그래프는 인접 행렬을 사용
- 트리를 인접 리스트로 사용하는 게 효율적
- 트리를 인접 행렬로 사용하면 메모리 낭비 심함. 자식과만 간선이 존재하기 때문
