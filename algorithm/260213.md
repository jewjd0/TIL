# 최소 신장 트리
## 개요
### 최소 신장 트리란?
- 신장 트리(Spanning Tree): n개의 정점으로 이루어진 무향 그래프에서 n개의 정점과 n-1개의 간선으로 모든 정점을 연결하는 트리
- 최소 신장 트리(Minimum Spanning Tree): 무향 가중치 그래프에서 신장 트리를 구성하는 간선들의 가중치 합이 최소가 되는 신장 트리
- 그래프에서 최소 비용 문제
  - 모든 정점을 연결하는 간선들의 가중치의 합이 최소가 되는 트리
  - 두 정점 사이의 최소 비용의 경로 찾기
- 인접 리스트로 구성 가능
- 노드와 노드를 오가는 최단 거리를 구하는 게 아님. 전체적인 간선의 최소 비용을 구하는 것
## KRUSKAL 알고리즘
- 간선을 하나씩 선택해서 최소 신장 트리를 구성하는 방법
- greedy한 방법
1. 최초, 모든 간선을 가중치에 따라 오름차순 정렬
2. 가중치가 가장 작은 간선부터 선택
  - 선택 간선의 두 정점에 대해 상황에 따라 진행 
    1. 두 정점의 대표자가 다른 경우: 간선을 선택하고 두 집합을 하나로 합침
    2. 두 정점의 대표자가 같은 경우: 간선을 선택하지 않음(사이클 발생 방지)
3. n-1개의 간선이 선택될 때까지 2번 과정 반복 
## PRIM 알고리즘
- 하나의 정점에서 연결된 간선들 중에 하나씩 선택하면서 MST를 구성하는 방법
1. 임의 정점을 하나 선택하여 시작
2. 해당 정점과 연결된 모든 간선을 우선순위 큐에 삽입
3. 우선 순위 큐에 간선이 남았다면
   1. 가장 가중치가 작은 간선 선택, 도착 정점이 방문한 적 없는 정점이면
   2. 이 간선을 최소 신장 트리에 추가, 방문 표시
   3. 그 정점에서 방문 가능한 간선을 모두 우선순위 큐에 삽입
4. 우선순위 큐가 빌 때까지 3번 과정 반복

### KRUSKAL과 PRIM 비교
|구분|KRUSKAL|PRIM|
|--|--|--|  
|접근 방법|간선 중심|정점 중심|
|자료구조|서로소 집합|우선순위 큐|
|적합한 그래프|간선이 적은 희소 그래프|간선이 많은 밀집 그래프|
- 간선 정보가 리스트 형태로 주어짐 -> KRUSKAL
- 인접 리스트 형태로 주어짐 -> PRIM
- kruskal은 간선 개수에 영향을 많이 받음
- prim은 정점 개수에 영향을 많이 받음
