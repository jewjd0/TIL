# 문자열 매치
## 개요
- 문자열 매치: 문자열에서 특정 패턴이 존재하는지 확인하는 문제. 예) 문자열 "hello world"에서 패턴 "world"가 존재하는지 확인
- 알고리즘: 브루트 포스, KMP 알고리즘, 라빈 카프 알고리즘, 보이어-무어 알고리즘
  - 보이어 무어 알고리즘은 파이썬 in 연산자에서 사용되는 알고리즘
## 브루트 포스 알고리즘
- 본문 문자열을 처음부터 끝까지 차례대로 순회, 패턴 내의 문자들을 일일이 비교하는 방식으로 동작
- 시간 복잡도: O(n*m) (n: 본문 문자열 길이, m: 패턴 문자열 길이)
## 라빈 - 카프 알고리즘
- 문자열 검색을 위해 해시 값 함수 사용
- 문자 일일이 비교하는 대신, 패턴의 해시 값과 본문 안의 하위 문자열의 해시값만 비교
- 최악의 경우 O(n*m) (해시 충돌이 많이 발생할 때), 평균적으로는 선형에 가까운 O(n+m)
- 한글자씩 이동하여 길이만큼 읽는 게 아닌 새로 추가되는 문자, 그 전에 읽었던 문자를 이용해 해시 값 갱신
- 해시 충돌이 발생할 수 있기 때문에, 해시 값이 일치하더라도 실제 문자열을 비교하여 패턴이 일치하는지 확인해야 함
## 보이어 - 무어 알고리즘
- 패턴을 오른쪽에서 왼쪽으로 비교
- 패턴의 오른쪽 끝에 있는 문자 불일치, 이 문자가 패턴 내에 존재하지 않는 경우 패턴의 길이만큼 이동
- 최악의 경우 O(n*m) (패턴이 본문에 거의 일치하는 경우), 평균적으로는 O(n/m), 평균적으로 가장 빠른 속도를 가지는 알고리즘
- 패턴의 오른쪽 끝에 있는 문자가 패턴 내에 존재하는 경우, 이 문자가 패턴 내에서 가장 오른쪽에 위치한 곳으로 이동
- 텍스트 문자를 전부 비교하지 않아 조금 더 빠르게 매칭 가능
## KMP 알고리즘
- Knuth-Morris-Pratt 알고리즘
- 불일치가 발생한 텍스트 문자열의 앞 부분에 어떤 문자가 있는지를 미리 알고 있으므로, 불일치가 발생한 앞 부분에 대해 다시 비교하지 않음
- 패턴 전처리하여 부분일치 테이블 배열 생성, 잘못된 시작 최소화
  - pi[k] : 처음부터 k번째까지의 접두사와 접미사가 일치하는 최대 길이
- 시간 복잡도: O(n+m) (n: 본문 문자열 길이, m: 패턴 문자열 길이)
- 패턴의 0번째 인덱스를 제외한 각 인덱스마다 맨 앞부터 해당 인덱스까지 접두사가 접미사가 일치하는 최대 길이 계산하여 pi 배열에 저장
- 본문 문자열과 패턴 문자열을 처음부터 비교, 불일치 발생 시 pi 배열을 참조하여 패턴 문자열의 어느 위치로 이동할지 결정
