# 탐욕 알고리즘
### 개요
- 최적해 구하는 데 사용되는 근시안적 방법
- 최적화 문제(Optimization Problem)란 가능한 해들 중에서 가장 좋은 해를 찾는 문제
- 복잡한 전체 상황 고려보단, 현재 당장 유리해 보이는 선택지를 골라 나가는 전략
- 여러 경우 중 그 순간에 최적이라고 생각되는 것을 선택해 나가는 방식
- 지역적으로 는 최적, 최종적인 해답이 최적이라는 보장은 없음
- 한번 선택한 것 번복 없음 -> 대부분 탐욕 알고리즘은 단순, 제한적 문제에 적용
- if문 겁나 써서 풀었으면 어느 부분이 쓸모없는 조건이고 어느 부분이 중복 조건인가, 좀더 최적은 없는가 잘 생각해보기. 
  - 다 해보고 ai써서 리팩토링 부탁하는 것도 방법
- 처음 greedy 공부하면 코드가 드러워짐
  - 조건식이 많아지고, 중복되는 부분도 많아짐
  - 매 순간 최고의 선택을 하는 하나의 식 만들어서 쭉 사용할 수 있어야 함
  - 현재 해를 구하기 위해 앞의 해가 필요한 경우는 greedy가 아님. 
## 활용

### 배낭 짐싸기
- 물건들 집합
- 유형
  - 0-1 배낭 문제: 각 물건을 하나씩만 넣을 수 있음
  - factional knapsack
- 남은 무게에 비해 앞으로 남은 게 너무 많음
- 무게 대비 가치 비율 높은 것부터 넣기
- -> 최적해 안나옴
- 안된다 싶으면 다른 알고리즘 쓰기
### 활동 선택 문제
- 회의 시작 시간과 종료 시간 주어졌을 때 가능한 많은 회의가 열리게 하기
- 끝나는 시간 기준 오름차순 정렬
- 가장 빨리 끝나는 회의 선택, 선택한 활동의 종료시간보다 빠른 시작 시낙 가진 활동 무시, 같거나 늦은 시작시간 선택
## 탐욕 알고리즘의 필수 요소
- 탐욕적 선택 속성(Greedy choice property)
  - 탐욕적 선택은 최적해로 갈 수 있음을 보여라
  - 탐욕적 선택은 항상 안전
- 최적 부분 구조(Optimal substructure)
  - 최적화 문제 정형화
  - 하나의 선택을 하면 풀어야 할 하나의 하위 문제 남음
- 원문제의 최적해 = 탐욕적 선택 + 하위 문제의 최적해 임을 증명하라
