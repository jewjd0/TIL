# 정렬
- 파이썬의 sorted를 쓰지 않고 직접 구현했을 때의 이점이 무엇일까
## 버블 정렬
- 인접한 두 개의 원소를 비교하여 자리를 계속 교환하는 방식
- 과정
  1. 첫 번째 원소부터 인접 원소끼리 계속 자리를 교환하며 맨 마지막 자리까지 이동
  2. 한 단계가 끝나면 가장 큰 원소가 마지막 자리로 정렬
  3. 교환하며 자리를 이용하는 모습이 물 위에 올라오는 버블과 같다고 해서 버블 정렬이라고 불림
- 시간복잡도: O(n^2)
## 삽입 정렬
- 리스트를 정렬/미정렬로 분리, 미정렬 부분에서 원소를 하나씩 꺼내 정렬 부분에 알맞은 위치에 삽입하는 방식
- 과정
  1. 첫 번째 원소는 정렬된 부분으로 간주
  2. 두 번째 원소부터 시작하여 미정렬 부분에서 원소를 하나씩 꺼내 정렬된 부분에 알맞은 위치에 삽입
  3. 정렬된 부분이 하나씩 늘어나면서 전체 리스트가 정렬될 때까지 반복
  4. 미정렬 리스트가 빌 때까지 반복
- 시간복잡도: O(n^2)
- 파이썬에서 굳이 리스트 2개로 구현할 필요는 없고 앞부분을 정렬된 부분으로 간주하여 구현할 수 있음
- 코드가 버블 정렬과 거의 비슷
- 버블 정렬과 삽입 정렬은 느리지만 정확하고 구현이 간단함
## 카운팅 정렬
- 항목들의 순서를 결정하기 위해 집합에 각 항목이 몇 개씩 있는지 세는 작업을 하여, 선형 시간에 정렬하는 효율적인 알고리즘
- 제한사항
  - 정수, 정수 표현 가능한 자료만 적용 가능
  - 공간 확보를 위해 집합 내의 가장 큰 정수를 알아야 함
- 시간 복잡도: O(n+k) (n: 입력 리스트의 길이, k: 집합 내의 가장 큰 정수)
## 선택 정렬
- 포켓볼 순서대로 정렬하기
- 주어진 자료들 중 가장 작은 값의 원소부터 차례대로 선택하여 위치를 교환하는 방식
- 과정
  1. 주어진 리스트 최소값을 찾는다
  2. 맨 앞에 있는 값과 교환
  3. 맨 앞을 제외한 나머지 리스트에서 최소값을 찾아 맨 앞에서 두 번째 위치에 있는 값과 교환
  4. 반복
- 시간복잡도: O(n^2)
## 병합 정렬
- Merge Sort
- 여러 개의 정렬된 자료의 집합을 병합하여 한 개의 정렬된 집합으로 만드는 방식
- 분할 정복 알고리즘 활용
  - 자료를 최소 단위 문제까지 나눈 후 차례로 정렬
  - top-down 방식
- 시간복잡도: O(n log n)
- 과정
  1. 리스트를 반으로 나눈다
  2. 왼쪽과 오른쪽 부분 리스트를 재귀적으로 병합 정렬한다
  3. 두 부분 리스트를 다시 하나의 정렬된 리스트로 병합한다
## 퀵 정렬
- Quick Sort
- 주어진 배열을 두 개로 분할하고, 각각을 정렬
- 분할할 때 기준 아이템(pivot)을 중심으로, 작은 아이템은 왼쪽, 큰 아이템은 오른쪽으로 분할
- 시간 복잡도: O(n log n) (평균), O(n^2) (최악)
- 과정
  1. 리스트에서 하나의 원소를 선택하여 피벗으로 삼는다
  2. 피벗보다 작은 원소들은 왼쪽에, 큰 원소들은 오른쪽에 위치하도록 리스트를 분할한다
  3. 왼쪽과 오른쪽 부분 리스트를 재귀적으로 퀵 정렬한다

### 파이썬 정렬 선택 기준
- 파이썬의 정렬은 timsort, 튜리스틱? 최적화만을 위해 만들어진 알고리즘
- 예외사항같은 거 여러가지 가지고 있어 때에 따라 필요한 알고리즘 선택
  - 정렬된 부분이 많으면 선택정렬
  - 정렬된 부분이 크면 병합정렬, 쪼개는 과정 속에 정렬이 되어 있으면 그걸 바로 선택정렬로 활용
- 최적의 상황에 맞는 알고리즘을 선택하기 때문에 어지간하면 파이썬의 timsort가 가장 빠름
- lambda나 reverse 옵션을 이용하여 sort함수를 가장 잘 활용하는 것을 연습하기
