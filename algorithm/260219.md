# 최단 경로
## 개요
### 정의
- 간선의 가중치가 있는 그래프에서 두 정점 사이의 경로들 중에 간선의 가중치 합이 최소인 경로
### 유형
- 단일 시작 정점, 다른 모든 정점까지의 최단 경로
  - 다익스트라 알고리즘 : 음의 가중치 허용 안함
  - 벨만-포드 알고리즘 : 음의 가중치 허용
- 모든 정점에서 모든 정점에 대한 최단 경로
  - 플로이드-워셜 알고리즘
## 다익스트라 알고리즘
- 가장 기본적
- 시작 정점에서 다른 모든 정점으로의 최단 경로를 구하는 알고리즘
- 시작 정점에서의 거리가 최소인 정점 선택해 나감
- 탐욕 기법 사용, mst의 프림 알고리즘과 유사
- 우선순위 큐 사용
### 동작 순서
- 현재 상황에서 가장 좋은 선택 반복, 최종 해답 구하기
  1. 시작 정점에서 각 정점까지 최단 거리 저장할 리스트 해답(몯든 거리 무한대, 시작 정점 0으로 설정)
  2. 시작 정점, 거리 우선순위 큐 삽입
  3. 가장 짧은 거리 가진 정점 추출, 추출 정점, 인접 정점 확인.
  4. 인접 정점까지 거리가 기존에 저장된 거리보다 작으면 거리 갱신, 우선순위 큐에 삽입
  5. 우선순위 큐가 빌 때까지 반복
### 단점
- 음의 가중치 있을 경우 무한루프 발생

## 벨만포드 알고리즘
- 시작 정점에서 다른 모든 정점까지의 최단 경로 구하는 알고리즘
- 음수 가중치를 갖는 간선이 있는 그래프에서도 동작
- 다익스트라 알고리즘과 달리, 탐욕 기법 대신 동적 프로그래밍(DP) 접근 사용
- 우선순위 큐 사용하지 않음
- 다익스트라보다 느리지만, 음의 가중치 허용
### 동작 순서
1. 시작 정점에서 각 정점까지 최단 거리 저장할 리스트 생성
   - 모든 거리 무한대, 시작 정점 0으로 설정
2. 모든 간선 반복 검사, 각 간선을 통해 더 짧은 경로가 발견되면 거리 갱신
3. 마지막 정점을 제외한 모든 정점에 대해 2번 반복(총 V-1회)
4. 한번 더 모든 간선 검사, 거리 갱신이 발생하면 음수 사이클 존재

## 플로이드-워셜 알고리즘
- 모든 정점 쌍 간의 최단 경로 구하는 알고리즘
  - 다익스트라, 벨만포드와는 달리, 정점과 정점 간 최단 경로
- 동적 프로그래밍 사용, 최단 경로 점진적 갱신
- 음수 가중치 있어도 정상동작 (음수 사이클 제외)
- 다익스트라 알고리즘과 시간 복잡도 동일, 구현 간단
- 느림(O(V^3)), 작은 그래프에 적합. 가장 편함!
### 동작 원리
- 동적 계획 알고리즘으로 모든 쌍 최단 경로 문제 해결하려면 부분문제들을 찾아야 함
- 갖고 있는 노드 개수만큼 경로 거리 갱신
- 그렇게 갱신되는 과정에서 중간 경로까지도 최단 경로 갱신됨
- 2차원리스트를 만들어서 최단 경로 저장
  - 인접 행렬 만드는 느낌으로 비용을 저장하는 느낌
