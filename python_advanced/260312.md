# RegEx(정규표현식)
## 개요
- Regular Expression(정규표현식) : 방대한 텍스트 속에서 원하는 데이터만 정교하게 걸러내고, 문자열 내 특정 패턴을 검색, 추출, 치환하는 기능
### 실무 적용 분야
- 입력 값 검증(Validation)
  - 이메일, 전화번호, 주민등록번호 형식 검사
  - 비밀번호 규칙(길이, 특수문자 포함 여부) 검증
- 텍스트 검색 (Search)
  - 문서나 로그 파일에서 특정 패턴 찾기
  - 대소문자 무시 검색, 단어 경계 기준 검색
- 데이터 추출(Extraction)
  - 날짜, URL, 숫자, 금액 등 특정 형식 데이터 추출
  - 로그에서 IP 주소, 에러 코드 등 필요한 정보만 추출
- 문자열 치환, 데이터 정제, 로그 분석 등에서 활용

## 're' 모듈
- 정규 표현식 패턴을 컴파일하고 실행하는 기능 제공
### 주요 패턴 매칭 함수
- re.match() : 문자열의 시작부터 패턴이 일치하는지 검사, 매치 객체 반환 
  - 엄격한 형식 검증
- re.search() : 문자열 전체에서 처음 발견된 매치 객체 반환 
  - 텍스트 내 특정 정보 존재 여부 확인
- re.findall() : 패턴과 일치하는 모든 부분을 리스트로 반환 
  - 데이터 수집 및 대량 추출
- re.finditer() : 패턴과 일치하는 모든 매치 객체를 반복자(iterator)로 반환 
  - 대용량 텍스트를 메모리 효율적으로 처리
### Match 객체
- match() 또는 search() 함수는 성공 시 매치 객체 반환
- 주요 메서드
  - .group() : 매치된 전체 문자열 혹은 특정 그룹 추출
  - .start() / .end() : 매치된 부분의 시작과 끝 인덱스 반환
  - .span() : 매치된 부분의 시작과 끝 인덱스를 튜플로 반환 (`(start, end)`)
### re.compile()
- 같은 패턴 반복하여 사용할 때 활용
- 패턴을 미리 객체로 만들어두면 매번 패턴 해석할 필요 없어 속도가 빠름

## Meta Character와 Raw String
### Meta Character
- 문자 형태를 컴퓨터가 이해할 수 있는 기호로 바꾼 것
| 메타 문자 | 의미 | 설명 |
|---|---|---|
| . | 문자 1개 | 줄바꿈 문자를 제외한 어떤 문자 하나와 매치하느 패턴(공백 포함) |
| \d | 숫자(digit) | 0-9 사이의 숫자 문자 한 글자와 매치하는 패턴 |
| \w | 단어(word) | 알파벳, 숫자, 언더바(_) 중 한 글자와 매치하는 패턴 |
| \s | 공백(space) | 공백 문자(스페이스, 탭, 줄바꿈 등) 한 글자와 매치하는 패턴 |
| \b | 단어 경계(word boundary) | 단어와 비단어 사이의 경계 (공백 아님, 2026-03-12같은 것) |
| \D | 숫자 아닌 것 | 숫자 문자가 아닌 문자 한 글자와 매치하는 패턴 |
| \W | \w 아닌 것 | word 문자가 아닌 문자 한 글자와 매치하는 패턴 |
| \S | 공백 아닌 것 | 공백 문자가 아닌 문자 한 글자와 매치하는 패턴 |
### Raw String
- Python 문자열의 escape(\n, \t 등) 를 특별히 해석하지 않고 백슬래시 그대로 문자열에 포함시키는 문자열 리터럴
  - Regex의 escape는 사용 가능
  - r'\d' -> 숫자 의미하는 메타 문자 표시
  - r'$' -> 문장 끝을 의미하는 메타 문자
  - r'\$' -> $ 문자 자체를 의미 (메타 문자 escape)
  - r'\\' -> 백슬래시 문자 자체를 의미 (파이썬 문자열 escape와 형태 동일)
- 문자열에 r이 접두사로 붙으면 Raw String을 의미
### Regex Pattern 예시
- Pattern은 특정한 규칙을 가진 문자열의 추상적인 형태 의미
  - 이메일 주소 : asdf@gmail.com, hello@ssafy.co.kr -> r'[\w\.]+@[\w\.]+'
  - 전화번호 : 010-1234-5678, 02-123-4567 -> r'\d{2,3}-\d{3,4}-\d{4}'

## Regex Syntax
### Quantifiers(반복자)
- 앞에 온 문자가 몇 번 나타나는지 지정할 때 활용
- 데이터 길이 제한 시 매우 중요하게 사용
| 반복자 | 설명 | 예시 |
|---|---|---|
| + | 1회 이상 (최소 하나) | r'\d+' -> 숫자가 1회 이상 반복되는 패턴 (123, 4567 등) |
| * | 0회 이상 (최소 0개) | r'\d*' -> 숫자가 0회 이상 반복되는 패턴 (빈 문자열도 매치) |
| ? | 0회 또는 1회 (있을 수도, 없을 수도 있음) | r'\d+ms?' -> 숫자가 1회 이상 반복되고 뒤에 m + (s가 있을 수도, 없을 수도) 패턴 (100ms, 200m 등) |
| {n} | 정확히 n회 반복 | r'\d{4}' -> 숫자가 정확히 4회 반복되는 패턴 (2024, 1234 등) |
| {n, m} | n회부터 m회까지 (n이상 m이하) | r'\d{2,4}' -> 숫자가 2회 이상 4회 이하 반복되는 패턴 (12, 123, 1234 등) |
### 문자 집합
- 대괄호([])로 표현, 집합에 포함된 문자 중 하나와 매치할 때 사용
- 쉼표나 공백 또한 하나의 문자로 인식, Python List와 혼동하지 말 것
| 사용 예시 | 의미 | 
|---|---|
| [abc] | a, b, c 중 하나와 매치 | 
| [a-z] | 소문자 알파벳 중 하나와 매치 |
| [A-Z] | 대문자 알파벳 중 하나와 매치 |
| [0-9] | 숫자 중 하나와 매치 |
| [a-zA-Z0-9] | 알파벳 대소문자와 숫자 중 하나와 매치 |
| [a, b, c] | a, b, c 중 하나와 매치 (쉼표 포함) |
| [가-힣] | 한글 글자 중 하나와 매치 |
### 앵커(Anchor)
- 문자열의 어디에 위치해야 하는지를 정의할 때 사용
- 데이터 파싱 시 줄의 시작이나 끝을 명확히 할 때 사용
| 앵커 | 의미 | 사용 예시 |
|---|---|---|
| ^ | 문자열의 시작 | r'^Hello.*' -> 문자열이 'Hello'로 시작하는 패턴 |
| $ | 문자열의 끝 | r'.+\.pdf$' -> 문자열이 '.pdf'로 끝나는 패턴 |

## Regex Group
### Syntax
- 소괄호(())를 사용하여 표현
- 역할
  - 패턴 범위 지정(grouping) : 여러 메타 문자를 하나의 단위로 묶는 역할
  - 데이터 추출(capturing) : 괄호 안에서 매칭된 내용을 기억하여 추출하는 역할
  - 그룹 재사용(backreference) : 앞서 정의한 패턴과 동일한 패턴을 다시 정의하지 않고 이전에 정의한 패턴을 참조하여 효율적으로 패턴 구성하는 역할
### Grouping
- 여러 Meta Character를 하나의 단위로 묶는 역할
- (pdf|csv|xlsx) -> pdf, csv, xlsx 중 하나와 매치하는 패턴
### 데이터 추출(Capturing)
- 괄호 안에서 매칭된 내용 기억하여 추출하는 역할
  - index기반 그룹 추출 방식(Positional Group)
    - 그룹 생성 순서대로 번호 부여
    - 번호는 1부터 시작, 0은 전체 매치된 문자열 의미 
  - 이름 붙인 그룹 추출 방식(Named Group)
    - 그룹에 이름 지정하여 이름으로 데이터 추출
    - 코드 가독성 좋음
    - (?P<그룹명>패턴) 문법 사용
    - 데이터 처리/로그 분석/파이프라인 작업에서 거의 표준처럼 사용
    - 여러 데이터 매칭 시 pandas dataframe 변환에 유리
### 그룹 재사용
- 앞서 정의한 패턴과 동일한 패턴 재정의 없이 이전에 정의한 패턴 참조하여 효율적으로 패턴 구성
- 문자열 구조의 일관성 검사 시 주로 활용, 자주 사용 안함

## Regex 응용
### Lookaround(전후방 탐색)
- 특정 단어 뒤에 오는 숫자만 필요할 때, 혹은 특정 단어는 매칭 결과에 포함하고 싶지 않을 때 사용
- 조건 찾아서 확인하지만 찾은 값 사용하지 않고 다른 값 필요할 때 활용
- 특정 단위나 레이블 기준으로 수치만 뽑을 때 유용
| 기법 | 문법 | 의미 | 예시 |
|---|---|---|---|
| 긍정 전방 | (?=패턴) | 뒤에 패턴이 있는 경우 앞의 내용과 매치 | r'\d+(?=원)' -> 뒤에 바로 원이 오는 숫자만(원화 숫자만) |
| 부정 전방 | (?!패턴) | 뒤에 패턴이 없는 경우 앞의 내용과 매치 | r'\d+(?!개)' -> 뒤에 바로 개가 오지 않는 숫자만(단위가 개가 아님) |
| 긍정 후방 | (?<=패턴) | 앞에 해당 패턴이 있는 경우 뒤의 내용과 매치 | r'(?<=#)\w' -> 바로 앞에 #가 있는 문자(알파벳, 숫자, 언더바)만 |
| 부넝 후방 | (?<!패턴) | 앞에 해당 패턴이 없는 경우 뒤의 내용과 매치 | r'(?<!\\)\w' -> 바로 앞에 백슬래시가 있는 문자 하나만 |
### Lazy Matching(게으른 매칭)
- Regex에서 수량자(*, +, {n,m})은 기본적으로 탐욕적으로 동작
  - 가능한 긴 문자열 매칭하려 함
  - 의도와 다르게 과도한 매칭 발생 가능
- 특정 구간만 짧게 끊고 싶을 때 게으른 수량자 (+?, *?, {n,m}?) 사용
  - 수량자 뒤에 ? 붙이면 가능한 적게 매칭
### Non-Capture Group
- (?:패턴) 문법으로 사용
- 패턴 논리적으로 묶지만 추출 결과 형태에는 영향 없음

## 참고
### Meta Character의 \b
- 단어의 경계
  - 단어 문자(\w)와 비단어 문자(\W) 사이의 경계 위치로 공백이 아님
  - 단어 단위로 정확히 찾고 싶을 때 주로 사용

- Groupping을 할 때 or 연산자 | 는 있지만 and 연산자는 없음

### 후방 탐색은 고정 길이 패턴만 사용
- 뒤로 확인하는 게 가변이면 범위 계산 불가

# Numpy
## 개요
### Numpy란?
- 파이썬에서 다차원 배열을 처리하기 위한 표준 라이브러리
- 대규모 다차원 배열과 행렬 지원
- 서로 다른 크기의 배열도 규칙에 따라 연산 확장
- 파이썬 리스트 대비 연산 속도가 빠르고 메모리 사용량이 적어 대규모 데이터 처리에 적합
### Python List의 한계
- 다양한 타입의 객체를 담을 수 있는 유연한 컨테이너지만, 대규모 수치 데이터 처리에는 비효율적
- 리스트는 객체 참조 방식으로 동작
  - 메모리 어딘가에 저장된 객체를 가리키는 주소값
  - 값 꺼낼 대마다 주소 찾아가서 객체 열어봐야 함
  - 주소가 사방에 흩어져 있어 캐시 효율성 떨어짐
- 리스트는 연산자 사용 시 수학적 계산이 아닌 리스트 조작으로 동작
- 모든 요소에 2를 곱하려면 
  - [1, 2, 3] * 2 -> [1, 2, 3, 1, 2, 3] (리스트 반복)
  - list(map(lambda x: x*2, [1, 2, 3])) -> [2, 4, 6] (map과 lambda로 요소별 연산)
  - 수학적으로 직관적이지 않고 느림
- 두 리스트의 요소를 더해서 하나의 리스트로 만들고 싶다면 반복문 필요
  - list comprehension도 반복 동작 수행
  - 차원 높아질수록 반복문 증가
### Numpy를 사용하는 이유
- 연속된 메모리 구조 사용하여 캐시 히트 효율 높음
- 단일 데이터 타입 사용으로 메모리 낭비 최소화
- C 기반 내부 구현으로 반복문 오버헤드 줄임
  - 대규모 데이터 처리 시 파이썬 리스트 대비 빠른 연산 속도 제공

## Numpy 기초
### Dimension
- 배열이 몇 개의 방향으로 구성되었는지 나타냄
### 축
- 배열의 각 차원이 뻗어 있는 방향
  - axis = 0 : 첫 번째 차원(배열 안 0번째 대괄호 묶음 방향)
  - axis = 1 : 두 번째 차원(배열 안 1번째 대괄호 묶음 방향)
  - axis = n : n+1번째 차원(배열 안 n번째 대괄호 방향)
  - axis = -1 : 마지막 차원
    - (1080, 1920, 3)이미지에서
    - axis = -1 -> RGB 채널 축(크기 3)
### Numpy Indexing
- 1차원에서는 List Indexing과 동일하게 동작
- 2차원에서는 [행, 열]순서로 접근
- 3차원에서는 [묶음, 행, 열] 순서로 접근
### Numpy Slicing
- 리스트 슬라이싱은 새로운 리스트를 복사해서 만들지만 Numpy 슬라이싱은 원본 배열의 뷰(view)를 생성
  - numpy slicing된 값 수정 시 원본 배열도 함께 변경
- 1차원 슬라이싱은 파이썬 리스트 슬라이싱과 동일
- 2차원 슬라이싱은 Indexing과 동일
  - [행, 열] 형태로 슬라이싱
  - ...(Ellipsis) : 다차원 처리 시 유용, 지정하지 않은 차원 전부 가져올 때 사용
  - list[:, 1:3] 같은 형식으로 사용, 3차원도 동일

## 벡터화와 브로드캐스팅
### 벡터화(Vectorization)
- 반복문을 명시적으로 작성하지 않고 배열 연산 수행
- 반복 작업을 C로 구현된 넘파이 내부 엔진에 전달하여 진행
  - offloading이라고 함
  - 오프로딩 : 특정 업무를 더 잘 처리할 수 있는 대상에게 해당 작업을 맡기는 것
### Numpy 벡터화와 Python 루프
- Numpy 벡터화
  - 동일한 타입의 데이터를 연속된 메모리에 저장
  - 타입 확인없이 배열 전체 한 번에 연산
- Python 루프 - 매 반복마다 아래 과정 반복
  - 인터프리터 코드 해석
  - 요소 타입 확인
  - 해당 타입 연산 가능 여부 확인
  - 연산 수행
### 브로드캐스팅
- 모양이 다른 두 배열 간의 산술 연산을 가능하게 하는 매커니즘
- numpy가 오른쪽 끝 차원부터 비교하여 부족한 차원 논리적 확장
  - 실제 메모리 복사 없이 확장된 것처럼 연산
### 브로드캐스팅 장점
- 불필요한 메모리 복사 없이 연산 가능 -> 메모리 효율적
- 루프 없이 배열 전체 연산 적용 -> 코드 간결
- 넘파이 내부 최적화 연산 활용 -> 속도 향상
### 브로드캐스팅 규칙
- 두 배열의 차원 수 맞춘다
  - 차원 수 다르면 부족한 쪽 앞(왼쪽)에 1 붙여 맞춘다
- 오른쪽(마지막 축)부터 같은 위치끼리 하나씩 비교
  - 크기 같으면 그대로 연산
  - 둘 중 하나가 1이면 1인 쪽 반복 연산(메모리 복사 없음)
  - 둘 다 1이 아니고 크기도 다르면 오류 발생
- 해당 규칙을 만족하지 못하면 브로드캐스팅 불가
### 브로드캐스팅 과정
- 두 배열의 shape 확인
- 크기 다른 두 배열 연산 시 브로드캐스팅 내부적 진행

## Numpy 주요 함수
### 주요 함수 분류 5가지
1. 수열 및 그리드 생성
2. 차원 조작 및 구조 설계
3. 조건부 선택 및 논리 연산
4. 기본 통계 연산
5. 특수 연산
### 공통 파라미터
- axis 
  - 연산 수행 축
  - None이면 전체 배열에 대해 연산
  - 정수 -> 해당 축 대상
  - 튜플 -> 여러 축 동시 대상
  - 연산 시 지정한 axis는 사라지는 축
- **keepdims**
  - True이면 연산 후에도 축 사라지지 않고 크가 1로 남음
  - 연산 후 축소된 축을 크기 1로 유지
  - False(기본값) -> 축 사라짐
  - 브로드캐스팅과 결합한 정규화 패턴에서 필소
- dtype
  - 연산 결과 데이터 타입 지정
  - 정수 배열 합산 시 오버플로 방지
  - 정수 배열을 실수로 합산할 때 정밀도 확보
### 수열 및 그리드 생성 관련 함수
- np.arange(start, stop, step)
  - 정수 수열 생성
  - np.arrange(1, 10, 2) -> [1, 3, 5, 7, 9]
- np.linspace(start, stop, num)
  - 특정 구간은 n등분한 좌표 생성(시각화, 함수 샘플링)
  - stop구간 포함, num은 등분 개수(기본값 50)
  - np.linspace(0, 1, 5) -> [0. 0.25 0.5 0.75 1.]
- **np.eye(N)**
  - 단위 행렬 행성 (선형 대수, 가중치 초기화)
  - np.eye(3)
### 차원 조작 및 구조 설계
- np.reshape(a, newshape)
  - 데이터를 유지하며 배열 구조 재정의
  - newshape : 변경할 shape(튜플)
  - -1 사용시 해당 차원 자동 계산
  - a.reshape(2, 3, 4)
  - a.reshape(2, -1)
- np.expand_dims(a, axis)
  - 브로드캐스팅 규칙을 맞추기 위해 차원 추가
  - axis : 차원 추가할 위치
  - np.expand_dims(a, axis=0) -> (1, 3, 4)
- np.squeeze(a, axis)
  - 크기가 1인 차원 제거
  - axis : 제거할 차원 위치 지정, None이면 모든 크기 1인 차원 제거
  - np.squeeze(a, axis=0) -> (3, 4)
- np.transpose(a, axes) 또는 a.T
  - 배열의 축 순서 변경
  - axes : 새로운 축 순서 지정하는 튜플, None이면 역순으로 변경
  - a.transpose(1, 0, 2) -> (3, 2, 4)
  - np.transpose(a, (2, 0, 1)) -> (4, 2, 3)
- np.stack(arrays, axis)
  - 여러 배열을 새로운 축으로 쌓아 고차원 데이터 생성
  - arrays : 쌓을 배열 리스트(모든 배열의 shape 동일해야 함)
  - axis : 새로운 축 추가 위치(기본 0)
### 조건부 선택 및 논리 연산
- np.where(condition, x, y)
  - 조건에 따라 두 배열에서 값 선택(삼항 연산자)
  - condition true시 x, false시 y
- np.isin(a, values)
  - 배열 요소의 포함 여부 배열 전체에 한번에 확인
  - boolean indexing과 조합하여 사용하기도 함
  - values : 포함 여부 확인할 값의 리스트
- np.any(a) / np.all(a)
  - 조건을 하나라도 만족하는지 / 모두 만족하는지 확인
  - axis : 검사할 축 (생략 시 전체)
### 기본 통계 연산
- np.sum(a, axis)
  - 배열 요소 합계 반환
  - axis : 계산할 축 (생략 시 전체)
  - np.nansum() : NaN무시하고 합계 계산
- np.mean(a, axis)
  - 배열 요소 평균 반환
  - axis : 평균 구할 축 (생략시 전체)
  - np.nanmean() : NaN 무시
- mp.median(a, axis)
  - 배열 요소의 중앙값 반환(이상치에 강건)
  - axis : 계산 축 (생략 시 전체)
  - np.nanmedian() : NaN 무시
- np.std(a, axis)
  - 배열 요소 표준편차 반환(표준편차 제곱 : 분산)
  - axis : 계산 축(생략 시 전체)
  - np.nanstd() : NaN 무시
- np.percentile(a, q, axis)
  - 배열에서 q번째 백분위수 반환
  - q : 백분위수(0-100) / axis : 계산 축(생략 시 전체)
  - np.nanpercentile() : NaN 무시
- NaN이 포함된 배열에서 일반 통계 함수 사용 시 결과가 NaN으로 반환
  - np.nan*() 함수 사용하여 NaN 무시하고 계산 가능
### 특수 연산
- np.unique(a, return_counts) 
  - 배열에서 중복 제거한 고유값을 정렬하여 반환
  - return_counts=True -> 고유값별 개수도 함께 반환
- np.argmax(a, axis) / np.argmin(a, axis)
  - 배열에서 최대값/최소값의 인덱스 반환
  - axis : 계산 축 (생략 시 전체)
- np.argsort(a, axis)
  - 정렬 순서 인덱스 반환
  - fancy indexing 활용하면 다른 배열 재배치에 사용 가능
  - kind : 정렬 방식 지정 (quicksort(빠르지만 불안정), mergesort(안정 정렬), heapsort(메모리 효율적))
- np.clip(a, a_min, a_max)
  - 배열 요소를 a_min과 a_max 사이로 제한
  - a_min보다 작은 값은 a_min으로, a_max보다 큰 값은 a_max로 변경

## Indexing Advanced
### Fancy Indexing
- 정수 리스트를 인덱스로 전달해 원하는 위치의 요소 추출하는 방식
  - 데이터 재배열할 때 유용하게 사용 가능
  - 데이터 셔플링 : 학습 데이터 무작위 순서로 섞을 때
  - 배치 샘플링 : 전체 데이터에서 특정 인덱스의 샘플만 뽑을 때
  - 특정 클래스 추출 : 레이블 배열에서 원하는 클래스의 데이터만 선택할 때
- 1차원 : 배열[[인덱스1, 인덱스2, ...]] (배열 내부 배열)
- 다차원 : 배열[[행1, 행2, ...], [열1, 열2, ...], ...]
### Boolean Indexing
- True/False로 이루어진 배열(마스크)을 인덱스로 전달해 조건에 맞는 요소만 추출
- 배열[ 조건식 ]
- 동작 순서
  1. 조건식 실행되어 true/false배열 생성
  2. true인 위치의 값만 원본 배열에서 추출
- arr[arr > 3]
- 다중 조건 문법
  - 일반적 and, or, not 키워드 numpy 조건식에서 사용불가
  - &(And), |(Or), ~(Not) 사용
### Fancy / Boolean Indexing 주의사항
- 인덱스 범위 초과
  - 배열 크기 벗어난 인덱스 전달 시 indexError 발생
- mask 크기 불일치
  - 마스크 크기가 원본 배열 해당 축 크기와 정확히 일치 필요
  - 마스크 크기가 작으면 ValueError, 크면 IndexError
- 복사본 생성으로 인한 메모리 사용
  - 두 방식 모두 원본 배열 크기만큼 메모리 추가 필요
  - 대용량 데이터는 슬라이싱(View) 활용 권장
- 다차원 팬시 인덱싱 결과의 shape
  - 전달하는 인덱스 배열의 shape에 따라 결과 shape 결정
  - arr[[0, 1], [0, 2]] -> (0,0), (1,2) 위치만 추출 -> shape(2,) 1차원 반환
  - 2차원 구조 유지 필요시 np.ix_() 사용
    - a[np.ix_([0, 2], [1, 3])] -> [(0, 1) (0, 3)], [(2, 1) (2, 3)] 형태 추출
    - shape(2, 2) 2차원 반환

## 참고
### Numpy와 Pandas
- Numpy는 오직 숫자와 위치에만 집중
  - 고차원 데이터도 메모리상의 연속된 숫자 배열로 표현
  - axis=0, 1, 2를 통해 어느 방향으로든 슬라이싱과 집계 빠르게 수행
  - 스트라이드 값 변경만으로 메모리 효율적 관리
- Pandas의 핵심 철학은 이름 있는 행과 열을 가진 표(Table)
  - 2차원까지 가로, 세로 축에 이름 붙여 관리하기 직관적
  - 3차원 구조 있었으나 시각화 한계/성능 저하로 공식적으로 삭제
- Numpy 유리한 경우
  - 숫자 계산 많고 빠른 처리 필요
  - 3차원 이상의 복잡한 다차원 데이터 다룰 때
  - 데이터 양 많아 메모리와 속도 중요할 때
- Pandas 유리
  - 표처럼 행과 열로 된 데이터 다룰 때
  - 빠진 값 정리나 데이터 손질 필요할 때
  - 날짜, 문자 등 다양한 타입 데이터 분석할 때
- **두 도구의 특징 이해하고 상황에 맞게 선택하는 것이 데이터 분석의 첫걸음**
