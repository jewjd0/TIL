# 이미지 생성 및 평가와 모델 학습
## 디퓨전 모델
- 처음부터 그림을 그리는 게 아닌, 노이즈로 가득한 랜덤 이미지에서 시작해 점진적으로 노이즈를 제거하면서 텍스트 조건에 맞는 이미지를 생성하는 방식.
  - 대리석에서 조각상 깎는 것과 비슷
- 노이즈 : 무작위 점의 집합. 가우시안 노이즈
- 디노이징 : 텍스트 프롬프트를 조건으로 반복적으로 노이즈 제거하는 과정
### SANA
- NVIDIA NVlabs에서 개발한 경량 고해상도 이미지 생성 모델
## HuggingFace Hub와 파이프라인
- HuggingFace Hub : 다양한 모델과 데이터셋을 공유하는 플랫폼. 앱스토어 같이 모델 ID만 알면 모델 가져다 쓰기 가능
- 파이프라인 : 모델을 쉽게 사용할 수 있도록 래핑한 인터페이스. 텍스트에서 이미지 생성, 이미지에서 텍스트 생성 등 다양한 작업 지원 pipe(prompt) 형태로 입력받아 끝내는 편리한 도구
## 프롬프트 엔지니어링
- 생성 모델에서 생성 결과의 품질 좌우하는 핵심 입력
- 동일한 프롬프트로 여러 장의 이미지 생성할 때 torch.Generator에 서로 다른 시드 지정하면 다양한 결과 얻기 가능. 시드 같으면 동일 노이즈에서 출발, 거의 같은 이미지 생성

### 언패킹
- \* : 리스트나 튜플처럼 순서 있는 데이터 언패킹
- \*\* : 딕셔너리처럼 키-값 쌍으로 이루어진 데이터 언패킹. 키에 맞춰서 데이터 언패킹
  - 딕셔너리에서 \* 하나만 쓰면 키만 언패킹, \*\* 쓰면 키-값 쌍 모두 언패킹

### 이미지 생성
- `num_images_per_prompt` : 프롬프트별 생성 이미지 수. gpu메모리 사용량 올라감. 메모리 부족하면 prompt 줄이고 반복문을 돌리기

### KD에서 student 모델의 학습률
- 선생 모델의 학습률을 0.01로 해서 학생 모델의 학습률을 0.005로 했더니 발산. 0.001로 더 낮추니 제대로 학습됨. 학생 학습률을 선생보다 훨씬 낮게 학습률을 조정하는 편이 좋은 것 같다
