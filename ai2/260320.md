### 데이터 전처리
- CIFAR-10 모델에 맞게 전처리
- transforms.Compose()로 여러 전처리 작업을 배열로 묶어 전달
```python
  transform = transforms.Compose([
      transforms.Resize((224, 224)),  # ViT 모델에 맞게 이미지 크기 조정
      transforms.ToTensor(),  # 이미지를 텐서로 변환
      transforms.Normalize(mean=[0.485, 0.456, 0.406], std=[0.229, 0.224, 0.225])  # 정규화

```
### Freezing
- 모델의 일부를 얼린다는 말은 특정 레이어의 가중치가 더 이상 변하지 않도록 고정하는 것
- requires_grad = False 처리
- 이미 검증된 보편적인 특징은 학습시킬 필요 없음
- 학습 속도 향상, 자원 절약, 과적합 방지
- 보통 초기 레이어는 얼리고 마지막 1~2개 레이어만 학습시킴
- Linear Probing : 모델의 마지막 레이어만 학습시키는 방법. Classifier 레이어만 학습하게 됨.
### 모델 학습
- 미니배치 단위로 데이터 가져와 반복
```python
for epoch in tqdm(range(num_epochs)):
    model.train() # 학습 모드 설정
    running_loss = 0.0
    for xb, yb in train_loader:
        xb, yb = xb.to(device), yb.to(device) # 데이터와 레이블을 GPU로 이동
        optimizer.zero_grad() # 옵티마이저의 기울기 초기화
        outputs = model(xb) # 모델에 입력 데이터 전달하여 예측값 계산
        loss = criterion(outputs, yb) # 손실 함수 계산
        loss.backward() # 역전파 수행하여 기울기 계산
        optimizer.step() # 옵티마이저 업데이트 수행하여 가중치 조정
        running_loss += loss.item() # 배치 손실 누적
```
- 과제에서 자주 쓰는 구간이니 외워둘 것.
### 데이터 증강
- 모델이 데이터를 외우지만 하지 않도록 여러 변형을 주는 전략
- RandomCrop : 이미지에서 랜덤하게 일정 크기만큼 잘라냄. padding으로 가장자리 패딩 추가 가능. 위치 변화에 강해지기
- RandomHorizontalFlip : p로 확률 전달해 확률대로 이미지를 좌우반전. 대칭 이해하기
### 학습률이란?
- 가중치 업데이트 시 이동하는 거리의 크기 조절하는 하이퍼파라미터. 보폭(Step size)이라고도 불림
- 이번 실습에서는 초기 학습에서는 0.1 정도, 이후 파인 튜닝 시 미세하게 조정하기 위해 작게 0.005 정도로 설정함
- 전체 레이어를 한꺼번에 학습시킬 때 학습률 높으면 수치 폭발하거나 학습 불안정해질 수 있음. 낮추면 안정적이지만 느리게 수렴할 수 있음. 적절한 학습률 선택이 중요
- 정답 근처에 온 상태에서 보폭이 너무 크면 정답 지나치고 진동할 수 있음.
- `optimizer = optim.SGD(model.parameters(), lr=0.0005)` 에서 lr이 학습률
### ImageProcessor, Optimizer
- ImageProcessor는 이미지 전처리 담당
- Optimizer는 모델의 가중치 업데이트 담당

### RandomHoriziontalFlip
- 파라미터(p)로 확률 받기 가능
- 확률을 따라 이미지 좌우 반전하여 데이터 증강
- 사물 방향이 바뀌어도 동일하게 인식하도록 하기 위함.
- 테스트 데이터 의존도 줄이는 효과 

### optimizer = optim.Adam(model.parameters(), lr=0.001, betas=(0.9, 0.999))
- betas : Adam이 과거의 정보를 얼마나 기억할지 결정하는 하이퍼파라미터
- beta1 : momentum(방향 유지), 0.9정도로 설정
  - 경사가 완만한 곳에서도 멈추지 않고 매끄럽게 이동하도록 도움. 급커브 방지
- beta2 : RMSProp(최근 변화량), 0.999정도로 설정
  - 변화가 가파른 곳에서는 변화 줄이고 변화 적은 곳에서는 변화 크게 하도록 해 안정적으로 최적점에 도달하게 함. 과속 방지.
