# 1. AI, ML, DL의 정의
### AI(Artificial Intelligence) 
- 주어진 환경/데이터를 인지, 학습, 추론을 통해 목표 달성 하도록 예측, 행동 선택, 계획하는 시스템
### ML(Machine Learning)
- AI 범주 내에서 데이터로부터 학습하여 목적을 달성하는 접근 방법론
- 예: 생성형 AI, 언어 모델, 이미지 분류 모델, 추천 시스템
### DL(Deep Learning)
- ML 범주 내에서 신경망(Neural Network) 함수를 사용한 학습 방법론

# 2. 데이터와 학습의 이해
## 1. 데이터 구성 요소(Feature Label)
### 데이터가 왜 중요한가?
- 머신러닝은 규칙 직접 코딩하지 않고, 데이터에서 규칙 학습
- 데이터(Feature, Label)의 분포 관계가 머신러닝의 학습 결과 결정
### Feateure(피처, 특성)
- 모델이 예측에 사용하는 입력정보
- 예측, 판단의 근거, 단서
### Label(라벨, 목표값)
- 모델이 예측하려는 정답
- 학습의 목표값 
## 2. ML 실생활 예시
### 예시 1. 유튜브 추천
- Feature: 영상 정보(장르, 크리에이터, 조회수, 좋아요 등), 사용자 정보(시청 이력, 구독 채널 등)
- Label: 영상에 대한 사용자 피드백(시청 여부, 좋아요 클릭 여부 등)
### 예시 2. 스팸메일 분류
- Feature: 메일 제목, 발신자, 단어 빈도 등
- Label: 스팸 여부(스팸, 정상)

# 3. 단일 피쳐 기반 학습
## 1. 1D 피쳐 기반 학습
### 1D 피쳐 기반 학습이란?
- 1D = 1차원
- feature가 하나인 경우, 머신러닝이 학습하는 가장 단순한 형태
### Income = f*(Years of Education) + epsilon
- 데이터셋 D : 30명의 years of education(피처)과 Income(라벨) 쌍
- D = {(x1, y1), (x2, y2), ..., (x30, y30)}
- x: years of education, y: income
- 미지의 함수 f*와 
  - feature와 label의 실제 평균 관계
  - 직접 관측 불가
  - 오차 포함 데이터(점)만 관측 가능
- 오차항 epsilon으로 표현
  - 데이터에는 주로 측정 오차 섞임
  - 원인 : 측정 기기의 한계, 환경적 요인 등
  - 데이터 = 참 함수 + 오차 (f* + epsilon)
### 피쳐와 라벨의 관계를 가장 잘 나타내는 함수 f는?
- 데이터를 설명하는 여러 함수 후보 존재
= 어떤 함수가 가장 잘 맞
는지 학습 필요

## 2. 모델과 가설공간
### 학습(Learning)
- 입력(Feature) -> 출력(Label)_ 관계를 찾는 과정
- 평균 관계를 하나의 함수로 표현
- 관계 표현 가가능한 함수는 무수히 많음
### 가설 공간(Hypothesis Space)
- 관계 표현 가능한 모든 후모 함수 모음
- 피처 공간과 라벨 공간 위에서 정의된 함수들 집합
### 모델(Model)
- 가설공간에 속한 특정 함수 f
## 3. 학습이란
### 학습
- 주어진 데이터에서 정답을 가장 잘 맞출 수 있도록 모델의 규칙을 조금씩 조정해가는 과정
- 데이터 D -> 가설 공간 H -> 모델 f 선택
### 학습에 필요한 3가지
1. 데이터
  - 학습 예시들 
  - 입력(특성)과 정답(라벨) 쌍으로 된 정답 모음
2. 가설 공간
  - 선택 가능한 모든 후보 함수들의 집합
  - 이 중에서 가장 좋은 함수 찾기
3. 선택 기준(손실 함수)
  - 예측값과 실제값 차이 측정하는 정량적 지표
### 학습 과정
1. 가성공간에서 함수 하나 선택
2. 그 함수로 데이터 모든 예시 예측
3. 손실함수로 틀린 정도 계산
4. 더 적게 틀리도록 파라미터 조정
5. 1-4 반복하여 최적의 함수 찾기
### 데이터 D -> 가설 공간 H -> 모델 f 선택 관점으로 생각
- 데이터가 제시한 문제를, 가설공간이 제공한 선택지 중에서 가장 잘 푸는 함수 찾기 = 학습

# 4. 복수 피쳐 기반 학습
## 1. 2D 피쳐 기반 학습
### Income = f*(Years of Education, Seniority) + epsilon
- 파란 surface(=미지의 참 함수 f) : 관측 불가
- 빨간 점(=데이터) : 관측 가능
- 학습 전 : 어떤 가상 공간을 사용할 것인가
- 학습 후 : 데이터를 활용하여 어떤 모델을 선택해야 할 것인가
- 선형 함수 : 데이터 차원보다 한 차원 낮은 함수 ex) 3차원 데이터 -> 평면 함수
- 모델 평면에서 떨어진 빨간 점 : epsilon 아님. 참 함수에서 떨어진 게 epsilon.
- epsilon : 데이터가 참 함수에서 떨어진 정도. 참 함수를 관측할 수 없기 때문에 epsilon도 관측 불가.
- 데이터가 평면에 너무 붙어도 안됨. 과적합하다고 판정된다.
## 2. 일반적 용어 정리 및 모델 가정
### Income = f*(Years of Education, Seniority) + epsilon -> Y = f*(X) + epsilon
- Income : 예측하려는 라벨(목표) 변수 -> Y
- Years of Education : 첫 번째 피쳐 -> X1
- Seniority : 두 번째 피쳐 -> X2
- 다른 i번째 피쳐 -> Xi
- 피쳐 벡터 -> X = (X1, X2, ..., Xn)
- 모델(함수형) : f* : Rp -> R, Y = f*(X) + epsilon
- 측정오차 epsilon : epsilon은 피처 X와 독립 및 E[epsilon] = 0 가정
## 3. 왜 f(x)를 학습하는가?
### 예측
- 잘 학습된 f가 있으면 새로운 X=x에서 반응/목표 Y 예측 가능
### 중요 특성 파악
- 피처들 중 어떤 특성이 Y를 설명하는 데 중요하고, 어떤 것은 덜 중요(무관)한지 알 수 있음
### 해석 가능성
- f의 복잡도에 따라 각 구성요소 Xi가 Y에 어떻게 영향을 미치는지(증감 방향, 민감도 등) 이해 가능

# 지도학습의 개념
## 지도학습의 정의와 특징
### 지도학습(Supervised Learning)이란?
- 정답 라벨이 있는 훈련 데이터를 사용해 예측 모델 학습하는 방법
- 목표 : 훈련 데이터로 학습한 모델이 새로운 데이터에서도 정확한 예측 수행
## 지도학습의 종류
### 회귀
- 예측하고 싶은 결과값이 숫자일 때 사용
- 예) 가격, 점수, 온도 등
### 분류
- 예측하고 싶은 결과값이 범주형일 때 사용
- 예) 스팸/정상, 질병 유무, 이미지 분류 등
## 지도학습 용어
### 특성(Feature, x)
- 예측에 사용되는 설명 변수
- 예측 함수 f의 입력값
### 라벨(Label, y)
- 맞춰야 하는 정답(실제값)
### 예측값(Prediction, y_hat)
- 학습 과정에서 모델이 내놓은 결과(숫자 또는 범주)
- y의 추정치
### 오류(Error, epsilon)
- 예측값과 라벨의 차이 : epsilon = y_hat - y
- 오류 != 측정 오차
- 예측값과 실제값의 차이(오류)를 줄이는 것이 지도학습의 목표
# 회귀(Regression)
## 회귀의 정의와 예시
### 회귀(Regression)란?
- 입력으로부터 숫자를 얼마나 정확히 예측하는지
- 특징 : 라벨 및 예측 모델의 출력은 연속적인 수치
## MSE
### MSE(Mean Squared Error, 평균제곱오차)란?
- 예측값과 실제값의 차이의 제곱의 평균
- 모델의 예측 정확도를 숫자로 측정하는 지표
- MSE = (1/n) * Σ(y_hat_i - y_i)^2
### MSE 단위가 제곱이라 해석이 어렵다면
- 원래 데이터와 같은 단위라 해석이 쉬운 RMSE(Root Mean Squared Error, 평균제곱근오차) 사용 가능
## R-squared(R², 결정계수)
- label(y)의 분산 중 feature(x)로 설명되는 비율
- R² = 1 - (MSE / Var(y))
- 평균만 쓰는 단순한 예측보다 모델이 실제값을 얼마나 더 잘 맞추는지를 0~1사이 값으로 나타냄
- R² = 0 : 모델이 평균 예측과 동일한 성능, R² = 1 : 모델이 완벽한 예측, R² < 0 : 모델이 평균 예측보다 못한 성능
- 음수도 나올 수 있다!
## MSE vs R²
- 둘 다 모델의 정확도를 평가하기 위한 정량 지표
| 항목 | MSE | R² |
|---|---|---|
| 정의 | 예측값과 실제값의 차이의 제곱의 평균 | 평균 대비 예측 모델 상대 성능 |
| 범위 | 0에서 무한대까지 | 음수에서 1까지 |
| 해석 | 절댓값(구체적 오차 크기) | 상대적 비율(성능 정도) |
| 좋은 값 | 0에 가까울수록 좋음 | 1에 가까울수록 좋음 |
| 장점 | 구체적 오차 크기 제공 | 직관적 해석, 모델 간 비교 쉬움 | 
| 단점 | 직관적 해석 어려움 | 구체적 오차 크기 모름 | 
### 각 지표 선택 기준
- MSE : 예측 오차의 절댓값이 중요한 경우, 모델의 구체적 성능 평가 필요할 때
- R² : 직관적 평가, 여러 모델 비교, 모델이 전반적으로 얼마나 좋은지 알고 싶을 때
- => 필요에 따라 상호 보완적으로 함께 사용

# 분류(Classification)
## 분류의 정의와 특징
- 정의 : 입력 특성(Feature)으로부터 출력이 범주값(카테고리)으로 나타나는 예측 문제
- 특징 : 범주 라벨(이진/다중)
## 분류 정확도
- label이 범주로 되어 있기 때문에 MSE, R² 같은 회귀 지표 사용 불가
### 분류 정확도
- 전체 예측 중에서 맞춘 예측의 비율
- Accuracy = (TP + TN) / (TP + TN + FP + FN)
### 정확도가 높은 모델이 항상 좋은 모델은 아니다!
- 불균형 데이터에서는 정확도가 높아도 실제로는 성능이 좋지 않을 수 있음
- 정확도는 단순히 맞춘 비율. 정확도만 보는 게 아닌 다른 지표도 함께 봐야 함
## 혼동 행렬
### 혼동 행렬
- 예측과 실제 값 사이의 관계 행렬 형태로 표현
- TP(True Positive) : 실제 양성, 예측 양성
- TN(True Negative) : 실제 음성, 예측 음성
- FP(False Positive) : 실제 음성, 예측 양성(오탐)
- FN(False Negative) : 실제 양성, 예측 음성(누락)
### 정밀도(Precision)
- 양성으로 예측한 것 중 실제 양성의 비율
- Precision = TP / (TP + FP)
### 재현율(Recall)
- 실제 양성 중에서 양성으로 예측한 비율
- Recall = TP / (TP + FN)
### F1 Score
- 정밀도와 재현율의 조화 평균
- F1 = 2 * (Precision * Recall) / (Precision + Recall)

# 학습의 목적
- 목적 : 훈련 데이터로 학습한 모델이 새로운 데이터에서도 정확한 예측 수행하게 하는 것
### 모델 성능 평가 방법
- 성능 평가는 학습에 사용되지 않은 데이터로 평가
- 훈련에서 성능이 아무리 좋아도 새로운 데이터에서 성능이 좋지 않으면 좋은 모델이 아님
- 일반화 오류의 최소화 지향
## overfitting
- 훈련 데이터의 우연한 패턴/잡음까지 외워서 훈련 데이터는 잘 맞지만 테스트에서는 성능이 떨어지는 현상
- 훈련 오류 극히 낮음, 테스트 오류 높음/요동
### 왜 좋지 않은가?
- 표본 의존/불안정 : 훈련 데이터는 일부 표본이라 잡음 포함 가능. 이 잡음에 과하게 맞춰 학습하면 샘플의 작은 변화에도 예측 크게 흔들림(분산 증가)
- 일반화 실패 : 보지 못한 데이터(테스트) 오류 커짐. 모집단 성능과 격차 벌어짐
### 오버피팅 != 분포 변화로 인한 에러 증가
- 분포 변화로 인한 오류 : 훈련 데이터 분포와 테스트 분포가 달라(환경, 계절, 센서 변경 등) 성능이 떨어지는 현상
- 분포 변화로 인한 에러 증가는 모델이 과적합하지 않아도 발생 가능
- 오버피팅이 발생하는 건 같은 분포에서 새로운 데이터에서도 성능이 떨어지는 것
## overfitting vs underfitting
### overfitting vs underfitting 균형 잡기
- 오버피팅 : 모델이 너무 복잡 => 잡음까지 학습(테스트 성능 나쁨)
- 언더피팅 : 모델이 너무 단순 / 학습이 완료되지 않음 => 중요한 패턴 놓침 (오류 큼)
### 해결 실마리
- 더 많은 데이터, 테스트 데이터를 활용한 모델 선정, 교차 검증

# 교차검증
## 테스트 성능 평가
### 훈련 오류 vs 테스트 오류
- 훈련 오류 : 모델을 학습시킨 같은 데이터에 다시 적용해 계산한 오류
- 테스트 오류 : 학습에 쓰지 않은 새 관측치에 대해 모델을 적용했을 때의 평균 예측오류
- -> 기출 잘 본다고 본 시험 잘보는 게 아님
- 보통 훈련 오류는 테스트 오류와 다름. 훈련 오류는 테스트 오류를 심하게 과소평가하는 경우 많음
- 훈련 오류는 계속 낮아짐, 테스트 오류는 u자형
- 목적 : 테스트 오류의 u자형의 바닥이 되도록 하는 적절한 모델 찾기
  - 너무 많이 외우지도(과적합), 너무 대충 공부하지도 않은(과소적합) 적당히 점수 잘 나오는 지점 찾기
### 테스트 예측 오류 계산
- 이상적 케이스 : 충분히 큰 별도 테스트 데이터셋 -> 현실에서 구하기 어려움
- 현실에서는 테스트만을 위한 데이터를 갖기에 데이터 자체가 부족할 수 있음
### 대안: 재표본화(Resampling)를 통한 테스트 오류 추정
- 데이터를 나눠 여러 번 훈련 -> 평가를 반복해 테스트 오류 가능
- 방법 : 검증셋(hold-out), K겹 교차검증(K-fold cross-validation)
- 장점 : 별도의 테스트 데이터 없이 데이터를 더 효율적으로 사용 -> 일반화 오차 추정

## 검증셋 접근
- 가용 샘플들을 무작위로 훈련셋과 검증셋으로 분할
- 훈련셋으로 모델 적합, 검증셋으로 예측 후 검증 오류 계산
- 검증 오류는 정량 반응:MSE, 범주 반응:오분류율(또는 F1-score) 을 측정
### 단점
- 어떤 표본이 훈련/검증에 들어가느냐에 따라 검증 기반 테스트 오류 추정치 가변적
- 검증 접근에서는 훈련셋만으로 모델을 적합 -> 전체 데이터 학습보다 성능 낮게 추정(테스트 오류 과대 추정)될 수 있음
## K-겹 교차검증
- 테스트 오류 추정의 표준적 접근
- 추정치는 모델 선택과 최종 모델의 테스트 오류 규모 파악에 활용
- 데이터 잔체를 크기 동일한 k개 폴드로 무작위 분할
- 폴드 k를 검증, 나머지 k-1개 폴드를 훈련에 사용
- k회 반복하여 각 폴드가 한 번씩 검증셋이 되도록 함, 이후 평균 오류로 테스트 오류 추정
- 각각의 폴드가 겹치지 않아야 함
### leave-one-out 교차검증
- k = n인 경우, 즉 각 폴드가 하나의 관측치로 구성된 경우
- 훈련셋이 n-1개 관측치로 구성되고, 검증셋이 하나의 관측치로 구성됨
- 장점 : 데이터 최대한 활용, 편향 없는 테스트 오류 추정
- 단점 : 연산량 많음
- -> 적절한 k를 선택하여 교차검증 수행하는 것이 일반적

# 비지도 학습
## 비지도 학습
- 레이블(정답) 없이 데이터의 구조, 패턴, 집단(잠재 서브그룹)을 찾아내는 학습
- 대표 과제 : 군집화(Clustering), 차원 축소(PCA), 밀도추정/이상치 탐지 등
- 정답 예측이 아닌 구조/요약/표현
### 핵심 질문
- 무엇을 비슷함/다름으로 볼 것인가 (거리, 유사도 선택)
- 전처리(스케일 표준화 등)를 어떻게 할 것인가
### 비지도 vs 지도 학습
- 지도 학습 : 입력+라벨로 모델 학습, 예측 정확도 평가
- 비지도 학습 : 입력만으로 모델 학습, 구조/패턴 발견 평가
## 클러스터링
- 데이터 안에서 하위 집단을 찾는 기법 총칭
- 집단 내부는 유사, 집단 간은 상이하도록 데이터 분할
- 유사/상이는 문제/도메인에 따라 달라짐
- 특성에 따라 클러스터 잘 안보이거나, 알고리즘마다 결과 다를 수 있음
### 예: 마케팅 세그멘테이션
- 시장 세분화 작엄 => 클러스터링
## K-means 클러스터링
### K-means 클러스터링
- 모든 데이터를 겹치지 않게 빠짐없이 정확히 하나의 군집에만 나누는 알고리즘
### 핵심 아이디어
- 좋은 군집화 = 클러스터 내부 변동이 작은 분할
- 목표 : 클러스터 거리,차이의 합이 최소가 되도록 클러스터 할당
### 알고리즘
1. 초기화 : 관측치들에 무작위로 k개의 클러스터 할당
2. 반복(할당이 더 이상 바뀌지 않을 때까지)
  1. 각 클러스터의 중심(centroid) 계산(특성 평균 벡터)
  2. 각 관측치를 가장 가까운 중심의 클러스터에 재할당(거리 예=유클리드)
### 알고리즘 특성
- 중심점과의 평균 거리가 계속 감소 -> 매 반복마다 결과 더 좋아짐
- 최적해를 항상 찾는 건 아님 -> 초기화에 따라 다른 결과 나올 수 있음
  - 지역 최소값으로 수렴해 지역 최소값 문제 발생 가능
- 초기화 중요 -> 여러 번 시도 -> 가장 좋은 결과 선택 권장
## 계층적 군집
- k-means는 클러스터 수 k를 사전에 정해야 하는 단점이 있음
- 계층적 군집은 클러스터 수를 사전에 정하지 않고 전체 구조를 덴드로그램으로 제공
- 이번 강의에서는 상향식(agglomerative) 다룸
### 계층적 군집 알고리즘
1. n개 관측에 대해 쌍별 비유사도 계산, 각 관측치를 하나의 클러스터로 시작
2. 반복
  1. i개 클러스터 간 쌍별 비유사도를 모두 계산해 가장 유사한 두 클러스터 병합
  2. 병합 후 남은 i-1개 클러스터 간 비유사도 업데이트
- 매 단계에서 클러스터끼리 병합 이루어짐 -> 1개의 단일 클러스터 될 때까지 반복
### 단점
- 매 단계에서 모든 클러스터 쌍 거리 계산 필요  -> 데이터 수 많은 경우 계산량 많음
### 링크 유형
- single link : 두 개 클러스터 내 데이터 쌍별 거리 중 최소값을 군집 간 거리로
- complete link : 두 개 클러스터 내 데이터 쌍별 거리 중 최대값을 군집 간 거리로
- average link : 두 개 클러스터 내 데이터 쌍별 거리의 평균을 군집 간 거리로
### 링크 유형에 따른 결과
- 같은 데이터라도 링크 선택에 따라 결과 달라질 수 있음. 다양한 종류의 링크 사용 권장
## 클러스터링 주의점
### 체크리스트
- 스케일링 : 표준화가 필요한가?
  - yes, 변수 단위 차이 영항 큼
- 몇 개의 클러스터가 적절한가?
  - K-means, 계층적 모두 어려운 문제, 합의된 정답 x
- 단일 시도가 아닌 다회 시도 권장
