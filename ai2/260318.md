# 선형 회귀
## 선형 회귀
- 입력 변수와 출력 변수 사이의 관계를 직선 형태로 근사하여 예측하는 통계적 방법
- 지도학습의 가장 기초가 되는 접근 중 하나
- 단순해 보이지만 개념적, 실무적으로 유용

## 단순선형회귀
- 한 개의 설명변수(X)와 하나의 반응변수(Y) 사이의 선형(직선) 관계를 찾는 방법
- 목표 : 데이터를 가장 잘 설명하는 직선을 찾아 예측(y_hat)에 활용
### 최소제곱법
- 실제 관측값과 예측값의 차이(잔차, residual)를 제곱해 합한 값(RSS, 잔차제곱합)을 최소화하는 방법
- 목표 : 데이터를 가장 잘 설명하는 직선을 찾기 위해 계수 B0(절편)과 B1(기울기) 추정
- 계수를 측정하기 위한 공식 : closed-form solution(공식으로 바로 계산 가능한 해) 존재
- B1 = (Σ((Xi - X_mean) * (Yi - Y_mean))) / Σ((Xi - X_mean)^2)
- B0 = Y_mean - B1 * X_mean
### 계수 해석
- coefficients(계수) : 각 독립 변수의 영향력 측정, 양수면 양의 영향, 음수면 음의 영향
  - 계수 크기가 영향력을 표현한다고는 할 수 없음. 계수의 크기는 독립 변수의 단위와 범위에 따라 달라질 수 있기 때문
- 선형관계가 뚜렷한지 아닌지는 p-value로 판단할 것. p-value가 0.05보다 작으면 선형관계가 있다고 판단할 수 있다.
## 다중선형회귀
- 독립 변수(설명 변수, feature)가 여러 존재할 때 사용하는 회귀 분석 기법
- 단순 선형 회귀는 하나의 변수만 고려하지만 다중 선형 회귀는 여러 독립 변수 동시 고려, 종속 변수와의 관계 구함
- 수식
  - Y = B0 + B1\*X1 + B2\*X2 + ... + Bn*Xn + epsilon
  - Y : 종속 변수(반응 변수)
  - X1, X2, ..., Xn : 독립 변수(설명 변수)
  - B0 : 절편, 
  - B1, B2, ..., Bn : 각 독립 변수의 계수
  - epsilon : 오차항, 모델이 설명하지 못하는 부분
- 해석 : 다른 변수를 고정한 채 Xi가 1 단위 증가할 때 Y가 B_i 단위 증가(또는 감소)한다고 해석
### 계수 추정과 예측
- 여러 변수로 반응 번수 Y를 동시에 예측하는 모형
- 예측값 : Y_hat = B0 + B1\*X1 + B2\*X2 + ... + Bn*Xn
- 추정 방법 : 실제 값과 예측값의 차이(잔차)를 제곱해 합한 값 최소화
- RSS가 최소일 때 얻어지는 계수가 최적 추정치
- -> 여러 입력 변수를 동시에 고려해 데이터와 가장 가까운 평면을 찾는 과정
### 계수 추정 수학적 유도
- 24p 참조. 글로 쓰기가 어려움
### 결과 시각화
- 3차원 그래프에 점을 찍으면 회귀 평면이 그려짐
## 선형회귀 주의사항
### 결과 검증 및 테스트 성능
- 훈련 데이터에서의 성능
  - 회귀식을 만들 때 최소제곱 해는 훈련 데이터만 보고 계산됨
  - 학습에 사용된 훈련 데이터에서는 적합이 잘 되어 있을 것
  - 이는 테스트 성능 과소평가할 가능성 높음
- 테스트 성능 평가 필요
  - 선형회귀도 변일반화 성능 확인하려면 훈련에 사용되지 않은 새로운 데이터에 적용 필요
  - 수 많거나 고차항 사용 시 과적합 문제 여전히 발생 가능
  - 검증/교차검증을 통해 적절한 적합 찾기 가능
### 선형 회귀를 통해 얻을 수 있는 것
- 이상적 상황: 변수들이 연관되지 않고 독립적 -> 계수 해석 명확
- 문제 상황: 변수 서로 연관시 -> 계수 추정 불안정, 해석 혼동 가능
- 주의: 관찰 데이터의 상관관계로 인과관계 주장해서는 안됨
  - 선형 관계를 보인다고 해서 인과 관계가 있는 건 아님
  - 예) 아이스크림 소비량과 상어에 물리는 사고 수

# 로지스틱 회귀
## 분류(Classification)
### 분류란?
- 정해진 범주(카테고리) 중 하나로 지정하는 것
- 범주형 변수 : 수치의 크고 작음이 아닌 유한한 범주로 표현. 색, 성별, 스팸메일 유무 등
### 분류 함수 목표
- 입력(X, 특성 벡터) : 눈동자 이미지, 이메일 제목/내용
- 출력(Y < C, 범주) : 눈동자 색 C={파랑, 갈색, 초록}, 이메일 스팸 여부 C={스팸, 정상}. 순서 없는 집합
- 목표
  - 분류 함수 f(x) 학습하여 입력 X가 속할 범주 예측
  - 범주의 직접 예측보다 각 범주에 속할 확률 P 추정하는 것이 더 유용할 때 많음.
## 분류 문제에 선형회귀는 가능한가?
### 이진 분류 문제
- 선형회귀는 선형함수 계산하는 문제, 예측 값이 제한된 값 갖게 못함.
- 선형회귀는 예측 확률이 0과 1 사이에 있도록 보장하지 않음. 예측값이 0보다 작거나 1보다 클 수 있음. -> 확률로 쓰기 부적절
### 다중 범주 분류 문제
- 선형회귀는 정수형 코딩에 따라 범주 간 순서와 동일한 거리 가정
- 범주 변수는 순서 없는 라벨이므로 부적절
## 분류 문제에서 적합한 모델
### 로지스틱 회귀
- 시그모이드 함수 사용, 0~1범위 내 확률값 예측 보장
- 순서 없는 범주를 확률로 직접 예측해 적합

## 로지스틱 회귀(Logistic Regression)
### 출력 범위가 0~1인 시그모이드 함수
- 기울어진 S자 형태의 곡선
- 모든 실수 z에 대한 출력값이 0과 1 사이에 있도록 보장
- z가 커질수록 출력값이 1에 가까워지고, z가 작아질수록 출력값이 0에 가까워짐
- y = 1 / (1 + exp(-z)) = exp(z) / (1 + exp(z))
### 로지스틱 회귀 모형식
- 시그모이드 함수 y = 1 / (1 + exp(-z))에 Linear Regression 식 z = B0 + B1x식을 대입하면 y = e**(B0 + B1x) / (1 + e**(B0 + B1x))가 됨
### 로지스틱 함수와 선형회귀 관계
- 로지스틱 함수는 선형 함수 내포
- 두 함수 간의 관계 해석하기 위해 Odds(오즈)와 logit(로짓 변환) 사용
### Odds(오즈)와 logit(로짓 변환)
- Odds(오즈) : 성공(y=1) 확률이 실패(y=0) 확률에 비해 몇 배 높은가
- logit(로짓 변환) : 오즈에 log를 취한 함수 형태
### 로지스틱 함수와 선형회귀의 관계
- 로지스틱 모형식은 선형 모형식과 시그모이드 함수의 결합. 
- 로짓 변환시 선형 회귀 모형식으로 표현 가능
## MLE 활용 모수 추정
### 우도(Likelihood)
- 선형회귀에서 현재 함수가 데이터와 오차가 적은지 평가하기 위해 평균 제곱 오차를 지표로 삼듯, 확률을 계산하는 함수를 평가하기 위해 우도를 지표로 삼음
- 우도 : 현재 확률 함수가 데이터를 얼마나 잘 설명하는지 나타내는 지표
  - 모델의 학습은 우도 값을 높여 최대화가 되도록 하는 것이 목표 -> MLE(Maximum Likelihood Estimation, 최대 우도 추정) 방법 사용
### 로지스틱 회귀 우도 최대화
- 로지스틱 회귀와 같은 이진 분류 문제에서 우도 최대화
- 곱으로 이루어진 함수의 경우 미분이 어렵기 때문에 양 변에 log를 취해 곱셉을 더하기로 변환시킨 뒤 log-likelihood 만들어 최대화
- log-likelihood : log(Likelihood) = Σ(yi*log(pi) + (1-yi)*log(1-pi))
- log L(b)를 미분해 도함수=0에 근접하도록 수치적(반복) 최적화를 통해 계수 b 추정
### log-likelihood로 변형하여 최대화한 값 구해도 괜찮은가?
- 로그함수는 단조 증가 함수 -> 최대화하는 값은 동일
- log-likelihood 최대화하는 값이 likelihood 최대화하는 값과 동일

# 신경망 적합
## 선형회귀 예시
### 손실함수
- 모델이 얼마나 잘못 예측하는지를 측정하는 함수. 작을수록 모델 정확함
### 선형회귀 학습
- 등고선 : 손실 함수 값 크기
  - 밝을수록 손실 높음, 어두울수록 적음
### 경사 하강법(Gradient Descent)
- 손실 함수의 값이 줄어드는 방향으로 파라미터 이동하는 과정
## 수학 리뷰
### 미분을 통한 기울기 이해
- 이차 함수로, 그래프는 위로 열린 포물선
- 미분 결과는 기울기, 기울기가 0인 지점이 최소값 위치
## 경사하강
- 기울기 계산 : 손실 함수 L의 파라미터 b에 대한 편미분 계산
- 파라미터 업데이트 : b = b - learning_rate * dL/db
- 반복 : 기울기 계산과 파라미터 업데이트 과정을 여러 번 반복하여 손실 함수 최소화
### 함수에 따른 최적화 난이도
- 손실 함수 모양에 따라 최적화 난이도 달라짐
- convex(볼록) 함수 : 하나의 전역 최소값 존재, 최적화 쉬움
- non-convex(비볼록) 함수 : 여러 개의 지역 최소값 존재, 최적화 어려움
## 확률적 경사 하강법
### 경사 하강법 단점
- non-convex 손실 함수에서 지역 최소값에 빠질 수 있음
- 매 스텝마다 전체 데이터 미분값 업데이트. 스텝별 계산양 많음
- => 전체 데이터 대신 무작위 데이터 샘플 사용 -> 확률적 경사 하강법(Stochastic Gradient Descent, SGD)
### 특성
- 국소 최솟값
  - 일부 배치로 기울기 계산해 노이즈 섞임
  - 노이즈가 오히려 지역 최소값 탈출 도와줌
- 노이즈가 있지만 타당한 업데이트
  - 미니배치의 기울기는 전체 기울기 아니지만 평균적으로 올바른 방향 가리킴
  - 점진적 최적점 방향으로 수렴
- 계산 비용 절감
  - 전체 데이터셋에서 작은 배치만 사용 -> 반복 당 연산량 적음
  - 큰 데이터셋에서 효율적 학습 가능
- 수렴 특성
  - full batch처럼 매끄럽게 수렴하지 않고, 무작위성 때문에 더 많이 진동하며 움직임
  - 전역 최솟값 근처의 좋은 해에 도달 가능. convex문제에서는 full batch 보다 늦을 수 있음
## 역전파
- 출력 오차를 기준으로 그래프를 거꾸로 따라가 연쇄법칙으로 각 노드(파라미터 포함)의 미분값 계산하는 절차
